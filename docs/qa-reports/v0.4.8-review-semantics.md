# v0.4.8 검토: RELAY 병렬 dispatch + control 채팅 follow-up (의미론 + UX)

## 발견 이슈

### **HIGH: 보고함 도착 안 함 (사용자 보고) — 근본 원인 규명**

**증상**: control → cfo 로 dispatch → cfo 응답이 cfo 폴더의 ChatView 에는 보이지만, control 폴더의 보고함 (inbox) 에 도착하지 않음.

**근본 원인**: `replyReceived()` 의 폴더 라우팅 로직에서 relay 경로와 일반 경로의 **수신자 인식 불일치**.

**흐름 분석**:

1. control (from=control) → cfo (to=cfo) 로 dispatch
2. cfo 는 reply 발행: from=cfo, to=control
3. `DispatchService.dispatchInternal()` 라인 116: `await observer.replyReceived(reply: reply, sourceFolderHint: nil)` 호출
4. `ControlTowerDispatchObserver.replyReceived()` 라인 103-120:
   - `sourceFolderHint` 가 nil 이므로 `reply.to` (= control) 로 folder lookup
   - control 이 합성 AgentID 가 아니므로 `agentToFolder` 에서 nil 반환 → fallback 으로 `reply.from` (= cfo) 사용
   - **결과: cfo 폴더에 기록됨** ❌ (의도: control 폴더에 기록되어야 함)

**v0.4.8 relay 경로에선 개선됨**:

- `dispatchRelaysInParallel()` → `observer.relayResultArrived()` (라인 177-182) → `makeRelayResultHandler()` (라인 132-150)
- `parentEnv.to` (= control) 로 명시적으로 폴더 lookup → 올바름 ✓

**해결책**: 일반 dispatch 경로도 relay 처럼 `relayResultArrived()` 패턴으로 통일하거나, `replyReceived()` 호출 시 `sourceFolderHint` 를 **항상** 제공.

### **MEDIUM: 메시지 순서 불일치 — control 채팅 UI 에서 arrival order 표시**

**문제**: RELAY_TO [cfo, cmo, cto] 순으로 발행 → 각 자식이 1초, 3초, 2초 후 응답

**현재**: `wireControlMainChatRelay()` 라인 71 `for await (target, reply) in group`:

- withTaskGroup 은 **completion 순서** 로 yield (cto → cfo → cmo 순서로 도착)
- control 채팅에 cto/cfo/cmo 순으로 follow-up bubble 추가
- **사용자가 발행 순서를 예측 불가**

**UX 영향**:

- 발행 순서가 비즈니스 로직이라면 (예: CFO → CMO → CTO 순 보고) 현재 구현은 혼란 야기
- 응답이 동시에 도착할 확률 높음 → 재현성 낮음

**권장**: `(target, reply)` 튜플 을 미리 수집 후 **발행 순서로 정렬** 후 append:

```swift
var results: [(AgentID, MessageEnvelope?)] = []
for await result in group {
  results.append(result)
}
// relays 의 발행 순서대로 정렬
let sorted = relays.compactMap { relay in
  results.first { $0.0 == relay.target }?.1.map { (relay.target, $0) }
}
for (target, reply) in sorted { ... }
```

**현상**: 라인 93 `let formatted = "✓ **\(from)**: \(body)"`

**문제**:

- markdown `**bold**` 처리 후 `: ` separator 이 가독성을 떨어뜨림 (특히 한국어에선 공백 부자연스러움)
- 아이콘 ✓ 만으로도 "완료" 전달 — 추가 텍스트 "**{Name}**: " 필요?

**권장 형식**:

```
✓ **CFO**
응답 본문 (별도 블록처럼)
```

또는 bullet list:

```
• CFO: 응답 본문
```

### **LOW: 자식 폴더 ChatView 에 control 메시지 혼입 가능성**

**현상**: `injectIncomingDispatch()` 는 모든 dispatch 를 자식 폴더에 표시 (라인 124-127):

```swift
messages.append(ChatMessage.user(prefix + request.body, at: request.createdAt))
```

**의도 명확화 필요**:

- 사용자가 자식 폴더 ChatView 들어가면, dispatch 받은 메시지가 "자신이 보낸 메시지" 처럼 `user` role 로 보임
- 이건 **자식이 control 에게 대화하는 형태**로 표시되므로 맞는 듯하지만, UI badge/timeline 에서 혼동 가능성
- 예: 자식 ChatView 에 "[Control] dispatch 내용" + 자신의 응답 이 연속되면 "자신이 컨트롤 에이전트와 대화하는 중" 으로 보임

**권장**: injectIncomingDispatch 에 "incoming dispatch" marker 추가:

```swift
messages.append(ChatMessage.user(
  "📥 \(prefix)\(request.body)",  // dispatch 아이콘
  at: request.createdAt
))
```
