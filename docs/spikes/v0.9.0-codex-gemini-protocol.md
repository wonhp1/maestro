## 1. Codex CLI

### 1.1 패키지 정보

| 항목                 | 값                                      |
| -------------------- | --------------------------------------- |
| **npm package**      | `@openai/codex`                         |
| **Version (latest)** | `0.125.0`                               |
| **License**          | Apache-2.0                              |
| **Binary**           | `codex`                                 |
| **Source**           | https://github.com/openai/codex         |
| **Install**          | `npm install -g @openai/codex`          |
| **Brew alt**         | `brew install --cask codex`             |
| **Maintainer**       | OpenAI 공식 (dylan-hurd-openai 외 12명) |

### 1.2 명령 구조

```
codex [OPTIONS] [PROMPT]                # 인터랙티브 (TUI)
codex exec [OPTIONS] [PROMPT]           # 비대화형 (Maestro 가 사용할 모드)
codex login                             # OAuth 로그인
codex logout                            # 인증 정보 제거
codex login status                      # 인증 상태
codex resume <SESSION_ID>               # 인터랙티브 세션 재개
codex exec resume <SESSION_ID> [PROMPT] # 비대화형 세션 재개
codex mcp                               # MCP 서버 관리
codex mcp-server                        # Codex 를 MCP 서버로 실행
codex apply                             # 마지막 diff 를 git apply
codex sandbox                           # sandbox 안에서 명령 실행
```

### 1.3 비대화형 모드 (`codex exec`) 핵심 옵션

| Flag                                         | 용도                                                   | Maestro 사용       |
| -------------------------------------------- | ------------------------------------------------------ | ------------------ |
| `[PROMPT]`                                   | 초기 prompt (없으면 stdin)                             | ✅ argv 또는 stdin |
| `--json`                                     | **JSONL stdout 출력**                                  | ✅ 핵심            |
| `-m, --model <MODEL>`                        | 모델 선택                                              | ✅                 |
| `-C, --cd <DIR>`                             | working directory                                      | ✅                 |
| `-o, --output-last-message <FILE>`           | 최종 메시지 파일에 저장                                | (선택)             |
| `--output-schema <FILE>`                     | JSON Schema 응답 강제                                  | (선택)             |
| `--ephemeral`                                | 세션 파일 저장 X                                       | (테스트용)         |
| `--skip-git-repo-check`                      | git repo 밖 실행 허용                                  | ✅                 |
| `--full-auto`                                | low-friction sandboxed 자동 실행                       | (선택)             |
| `--dangerously-bypass-approvals-and-sandbox` | 모든 confirm 우회                                      | (회피)             |
| `-s, --sandbox <MODE>`                       | `read-only` / `workspace-write` / `danger-full-access` | ✅                 |
| `-c, --config <key=value>`                   | 임시 config override                                   | ✅                 |
| `--ignore-user-config`                       | `~/.codex/config.toml` 무시                            | (선택)             |

### 1.4 세션 / Resume

- **Session 식별자**: `thread_id` (UUID 형식)
- **재개 명령**: `codex exec resume <UUID> [PROMPT]`
- **세션 저장 위치**: `~/.codex/` 또는 `~/Library/Application Support/com.openai.chat/codex-*`
- **최근 세션 자동 선택**: `codex exec resume --last`
- **--ephemeral**: 세션 저장 안 함 (Maestro 가 자체 세션 관리할 때 유용)

### 1.5 출력 Format (--json)

비인증 상태에서 캡처한 실제 JSONL:

```jsonl
{"type":"thread.started","thread_id":"019dd7cd-c11c-7e31-8179-f92e4f5ccefd"}
{"type":"turn.started"}
{"type":"error","message":"Reconnecting... 1/5 (...)"}
{"type":"turn.failed","error":{"message":"..."}}
```

**예상 event types** (인증 후 정상 응답 시):

- `thread.started` — 세션 시작 + thread_id (UUID)
- `turn.started` — 사용자 턴 시작
- `message` 또는 `delta` — assistant 응답 chunk (TBD: 인증 후 캡처 필요)
- `tool_use` — tool call (TBD)
- `tool_result` — tool 실행 결과 (TBD)
- `turn.completed` — 턴 종료
- `turn.failed` — 에러 + error.message
- `error` — 일반 에러

### 1.6 인증 흐름

| 방식                              | 명령                                 | 인증 정보 저장                        |
| --------------------------------- | ------------------------------------ | ------------------------------------- |
| **OAuth** (ChatGPT Plus/Pro 구독) | `codex login`                        | `~/.codex/` 내 (정확한 파일명 미확인) |
| **API key**                       | `codex login --with-api-key` (stdin) | `~/.codex/` 내                        |
| **Device auth**                   | `codex login --device-auth`          | (CLI-only 환경용)                     |
| **상태 확인**                     | `codex login status`                 | "Not logged in" / "Logged in as ..."  |
| **로그아웃**                      | `codex logout`                       | 인증 정보 삭제                        |

**중요**: `codex login` 은 ChatGPT Plus/Pro 구독자가 본인 구독 그대로 사용 가능. API key 는 별도 종량제 결제.

### 1.7 Config

- `~/.codex/config.toml` — TOML 형식
- `-c key=value` 로 런타임 override
- `--ignore-user-config` 로 무시

| 항목                 | 값                                            |

| **Version (latest)** | `0.40.0`                                      |

| **Maintainer**       | Google 공식 (google-wombot, ofrobots, mrdoob) |

### 2.2 명령 구조

```
gemini [options] [query..]              # 기본 (인터랙티브)
gemini -p "prompt"                      # 비대화형 (Maestro 사용)
gemini -r <name>                        # 세션 재개 (latest 또는 인덱스)
gemini --list-sessions                  # 세션 목록
gemini --delete-session <index>         # 세션 삭제
gemini mcp                              # MCP 서버 관리
gemini extensions                       # 확장 관리
gemini skills                           # 에이전트 skills 관리
gemini hooks                            # Hooks 관리
gemini gemma                            # 로컬 Gemma 모델 라우팅
```

### 2.3 비대화형 핵심 옵션

| Flag                                               | 용도                                    | Maestro 사용   |
| -------------------------------------------------- | --------------------------------------- | -------------- |
| `-p, --prompt <TEXT>`                              | 비대화형 prompt                         | ✅             |
| `-i, --prompt-interactive <TEXT>`                  | prompt 후 인터랙티브 진입               | (X)            |
| `-m, --model <MODEL>`                              | 모델 선택                               | ✅             |
| `-o, --output-format <text\|json\|stream-json>`    | **출력 format**                         | ✅ stream-json |
| `-r, --resume <name\|"latest"\|index>`             | 세션 재개                               | ✅             |
| `-y, --yolo`                                       | 모든 도구 자동 승인                     | (선택)         |
| `--approval-mode <default\|auto_edit\|yolo\|plan>` | 승인 모드                               | ✅ default     |
| `--skip-trust`                                     | 폴더 trust 우회                         | ✅             |
| `-w, --worktree`                                   | 새 git worktree 시작                    | (X)            |
| `-s, --sandbox`                                    | sandbox 실행                            | (선택)         |
| `-e, --extensions`                                 | 사용할 extension                        | (선택)         |
| `--include-directories`                            | 추가 workspace 디렉토리                 | (선택)         |
| `--policy` / `--admin-policy`                      | policy 파일                             | (선택)         |
| `--screen-reader`                                  | 접근성 모드                             | (X)            |
| `--acp`                                            | ACP 모드 (Agent Communication Protocol) | (TBD)          |

### 2.4 환경변수

| 환경변수                          | 용도                 |
| --------------------------------- | -------------------- |
| `GEMINI_CLI_TRUST_WORKSPACE=true` | 폴더 trust 자동 부여 |
| `GEMINI_API_KEY`                  | API key (OAuth 대신) |

### 2.5 세션 / Resume

- **Session 식별자**: `session_id` (UUID 형식, init event 에 포함)
- **재개**: `gemini -r <name|index>` 또는 `gemini -r latest`
- **세션 저장**: `~/.gemini/` 내 (정확한 위치 TBD)
- **세션 목록**: `gemini --list-sessions`

### 2.6 출력 Format (-o stream-json) — 실제 캡처

```jsonl
{"type":"init","timestamp":"2026-04-29T05:54:09.834Z","session_id":"c50f52a9-6d52-4bf5-95c0-bd21d2564be5","model":"gemini-3-flash-preview"}
{"type":"message","timestamp":"2026-04-29T05:54:09.836Z","role":"user","content":"say hi in 2 words"}
{"type":"message","timestamp":"2026-04-29T05:54:19.353Z","role":"assistant","content":"Hello there.","delta":true}
{"type":"result","timestamp":"2026-04-29T05:54:19.374Z","status":"success","stats":{"total_tokens":10401,"input_tokens":10356,"output_tokens":3,"cached":0,"input":10356,"duration_ms":9540,"tool_calls":0,"models":{"gemini-3-flash-preview":{...}}}}
```

**Event types**:

- `init` — 세션 시작 + session_id + model + timestamp
- `message` (role=user) — 사용자 prompt echo
- `message` (role=assistant, delta=true) — assistant 응답 chunk
- `tool_use` — tool call (TBD: 다음 캡처)
- `tool_result` — tool 실행 결과 (TBD)
- `result` — 최종 stats (총 토큰 / cached / duration / tool_calls / models)
- `error` — 에러 (TBD)

### 2.7 인증 흐름

| 방식                    | 흐름                                               | 인증 정보 저장                   |
| ----------------------- | -------------------------------------------------- | -------------------------------- |
| **Google OAuth** (자동) | 첫 실행 시 브라우저 자동 오픈 → Google 계정 로그인 | `~/.gemini/oauth_creds.json`     |
| **API key**             | `GEMINI_API_KEY` 환경변수                          | (저장 X — 환경변수 only)         |
| **Account info**        | (로그인 후)                                        | `~/.gemini/google_accounts.json` |
| **Installation ID**     | 첫 실행 시 자동 생성                               | `~/.gemini/installation_id`      |

**중요**: `gemini auth login` 같은 별도 명령 **없음** — 첫 비인증 호출 시 자동으로 OAuth 트리거.

### 2.8 Trust 정책

Gemini 는 `~/.gemini/trusted_workspaces.json` 또는 환경변수로 trust 폴더 관리:

- 미신뢰 폴더에서는 일부 기능 차단 (skip-trust 옵션 또는 환경변수 필요)
- Maestro 가 세션 spawn 시 `--skip-trust` 또는 `GEMINI_CLI_TRUST_WORKSPACE=true` 주입 필요

## 4. 어댑터 구현 시 고려사항

### 4.1 Codex 어댑터 (CodexAdapter)

**ClaudeAdapter 와 유사한 점**:

- ✅ 비대화형 모드 + JSONL 스트림 (NDJSON)
- ✅ Session UUID 기반 + resume 지원
- ✅ 모델 선택 가능 (`-m`)
- ✅ OAuth (구독 토큰 사용 가능)
- ✅ API key fallback

**다른 점**:

- 🔄 `codex exec` 가 별도 subcommand — argv 첫 번째에 `exec` 필수
- 🔄 Resume 도 `exec resume <UUID>` 형식 (`--resume` flag X)
- 🔄 Working dir 명시 필요 (`-C <DIR>`) — Claude 는 cwd 자동
- 🔄 Sandbox mode 옵션 (`-s read-only|workspace-write|danger-full-access`) — Maestro 정책 결정 필요
- 🔄 `--skip-git-repo-check` 권장 (git repo 밖에서도 동작하게)
- 🔄 stdin prompt 지원 (`-` 또는 자동) — 큰 prompt 안전

**Parser 패턴**: ClaudeStreamParser 와 거의 동일 패턴 (NDJSON → event), event type 만 다름:

- `thread.started` → AgentEvent.sessionStarted(threadId)
- `turn.started` → AgentEvent.turnStarted
- `message` → AgentEvent.assistantContent(...) 또는 .userMessage(...)
- `tool_use` → AgentEvent.toolCall(...) (TBD)
- `tool_result` → AgentEvent.toolResult(...) (TBD)
- `turn.completed` → AgentEvent.turnCompleted
- `turn.failed` / `error` → throw error

**예상 코드량**: CodexAdapter ~400줄 + CodexProfile ~30줄 + CodexStreamParser ~150줄 = **~580줄** (Claude 와 비슷)

### 4.2 Gemini 어댑터 (GeminiAdapter)

**ClaudeAdapter 와 유사한 점**:

- ✅ stream-json 출력 (Claude 와 같은 옵션 이름!)
- ✅ Session ID + resume
- ✅ Model 선택
- ✅ OAuth (Google 계정 자동)
- ✅ API key 환경변수 fallback

**다른 점**:

- 🔄 Resume 가 `-r <name|index|latest>` — UUID 가 아닌 이름/인덱스 기반
- 🔄 Trust 정책 — `--skip-trust` 또는 `GEMINI_CLI_TRUST_WORKSPACE=true` 필수
- 🔄 별도 auth 명령 X — 첫 실행 시 자동 OAuth 브라우저 띄움
- 🔄 Approval mode (`default|auto_edit|yolo|plan`) — Maestro 가 default 사용 (사용자 승인 필요)
- 🔄 Event type: `init` / `message` / `result` (Claude 의 `system.init` / `assistant` 와 다름)

**Parser 패턴**: GeminiStreamParser

- `init` → AgentEvent.sessionStarted(sessionId, model)
- `message` (role=user) → AgentEvent.userMessage(content)
- `message` (role=assistant, delta=true) → AgentEvent.assistantContent(content)
- `tool_use` → AgentEvent.toolCall (TBD)
- `tool_result` → AgentEvent.toolResult (TBD)
- `result` → AgentEvent.completed(stats)
- `error` → throw

**예상 코드량**: GeminiAdapter ~400줄 + GeminiProfile ~30줄 + GeminiStreamParser ~150줄 = **~580줄**

## 5. 발견된 quirk / 제약사항

### 5.1 Codex

- **인증 필수**: 비인증 시 401 에러 + 5회 재시도 → ~10초 후 실패. Maestro 가 사전 검사 필요 (검사 가능 메서드: `codex login status`).
- **Session 저장 위치 미확정**: `~/.codex/` 와 `~/Library/Application Support/com.openai.chat/` 두 곳 사용. 인증 정보 / 세션 정확한 파일명 spike 단계에서 미확인 — Phase 1 EnvironmentChecker 작성 시 검증 필요.
- **`exec` subcommand 분리**: ClaudeAdapter 의 `claude -p` 와 달리 `codex exec [PROMPT]` 형식. argv 빌드 로직 차이.
- **Sandbox 정책**: 기본 `workspace-write`. Maestro 가 사용자에게 노출할지 / 항상 `workspace-write` 강제할지 결정 필요.
- **Tool use event 미확인**: 인증 후 실제 tool call 응답 캡처 못함. Phase 2C (streaming) 작성 시 실제 호출로 확인 필요.

### 5.2 Gemini

- **Trust 정책 필수**: 미신뢰 폴더에서 `-y, --yolo` + `--approval-mode default` 가 무시되고 prompt → "current folder is not trusted" 에러. Maestro 가 spawn 시 `--skip-trust` 또는 `GEMINI_CLI_TRUST_WORKSPACE=true` 주입 필요.
- **Stream event 'message' 의 delta**: assistant chunk 가 `delta:true` flag 로 표시. Maestro parser 가 delta=true vs delta=false (final) 구분 필요.
- **Result event 의 stats**: 토큰 사용량 / 비용 / cached 정보 풍부 — Maestro UI 에 노출 가치 있음 (선택사항).
- **세션 식별자가 name 또는 index**: UUID 가 아니라 사람이 정한 이름 또는 정수. 재개 시 `gemini -r latest` 가 가장 안정적.
- **Tool use event 미확인**: Codex 와 동일 — Phase 2C 에서 확인.

### 5.3 공통

- **NDJSON line buffering**: 두 CLI 모두 line 단위 JSON 출력. ClaudeStreamParser 의 line-buffered 패턴 그대로 적용 가능.
- **Tool use / tool_result format**: 두 CLI 모두 spike 단계에서 실제 캡처 못함. 인증 + 실제 호출 필요한 phase 2B/2C 에서 확인.
- **에러 발생 시 exit code**: 두 CLI 모두 비-0 exit code + stderr 에러 메시지 + JSON 의 error event 모두 사용. parser 가 모든 source 검사 필요.

## 7. 핵심 결정 사항
